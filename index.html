What is HTML

HTML (HyperText Markup Language) is not a programming language, it is a markup language, used to tell your browser how to structure the pages you visit. It can be as complex or as simple as the web developer wants. HTML consists of a series of elements, which you can use to enclose, delimit or mark different parts of the content to make them appear in a certain way, or act in a certain way. Tags that delimit a piece of content can cause that content to link to another page, italicize a word, and so on.

Anatomy of an HTML element

Let's explore the paragraph element a bit, The main parts of our element are:

  • The opening tag: consists of the name of the element (in this case, p), enclosed in opening and closing angle brackets. This indicates where the element begins or begins to take effect; in this case, where the paragraph begins.
  • The closing tag: is the same as the opening tag, except that it includes a forward slash before the item name. This indicates where the element ends; in this case, where the paragraph ends. Not including a closing tag is a common beginner mistake, and can lead to strange results.
  • The content: is the content of the element, which in this case is only text.
  • The element: it is made up of the sum of the opening tag, the closing tag and the content.
  • empty elements:Not all elements follow the above pattern of an opening tag, content, and a closing tag. Some elements consist only of a single tag, which is usually used to insert/embed something in the document at the place it is included.

Anatomy of an HTML document

That wraps up the basics of individual HTML elements, but they aren't very useful on their own. Now we'll look at how individual elements are combined to form an entire HTML page:

<!DOCTYPE html>
<html lang="en">
<head>
<meta charset="UTF-8">
<meta name="viewport" content="width=device-width, initial-scale=1.0">
<http-equiv="X-UA-Compatible" content="ie=edge">
<title>document</title>
</head>
<body>
<p> this is my page </p>
</body>
</html>

Here we have some of the most important tags:

<!DOCTYPE html>
The doctype. In the mists of time, when HTML was young (about 1991/2), doctypes were meant to act as links to a set of rules that the HTML page had to follow to be considered good HTML, which could mean automatic error checking and other useful things

<html> </html>
This element wraps all the content on the entire page, and is sometimes known as the root element.

<head> </head>
This element acts as a container for all the stuff you want to include on the HTML page, that isn't the content you are showing to your page's viewers. This includes things like keywords and a page description that you want to appear in search results, CSS to style our content, character set declarations, and more. You'll learn more about this in the next article of the series.

<meta charset="UTF-8">
This element specifies the character set for your document to UTF-8, which includes most characters from the vast majority of human written languages. Essentially, it can now handle any textual content you might put on it. There is no reason not to set this, and it can help avoid some problems later.

<body> </body>
This contains all the content that you want to show to web users when they visit your page, whether that's text, images, videos, games, playable audio tracks, or whatever else.

Standard History

In 1989 there were two techniques that allowed linking electronic documents, on the one hand hyperlinks or links (hyperlinks or links) and on the other hand a powerful tag language called SGML. At the time, Tim Berners-Lee, who worked at the European Center for Nuclear Research CERN, revealed to the press that he was working on a system that will allow access to online files that worked on computer networks or electronic machines based on the TCP / IP protocol. Initially, it was developed so that information could be easily shared between scientists from different universities and research institutes around the world.

In the early 1990s, he finally defined HTML as a subset of the well-known SGML and created something even more valuable, the World Wide Web.

Pei-Yuan Wei introduced ViolaWWW, which would work in text mode and on a UNIX operating system.

Work to create a successor to HTML, called HTML +, began in late 1993. HTML + was originally designed to be a superset of HTML that allowed it to gradually evolve from the previous HTML format. The first formal HTML + specification was therefore given version number 2 to distinguish it from previous unofficial proposals. Work on HTML + continued, but never became a standard, despite being the base formally closest to the compositional aspect of current specifications.

The draft HTML 3.0 standard was proposed by the newly formed W3C in March 1995. Many new capabilities were introduced with it; for example, facilities to create tables, make the text flow around the figures and show complex mathematical elements. Although it was designed to be compliant with HTML 2.0, it was too complex to be implemented with the technology of the time, and when the draft standard expired in September 1995, it was abandoned due to lack of support from web browser manufacturers. HTML 3.1 was never officially proposed, and the next standard was HTML 3.2, which abandoned most of the new features of HTML 3.0 and instead adopted many elements initially developed by the Netscape and Mosaic web browsers. The possibility of working with mathematical formulas that had been proposed in HTML 3.0 became integrated into a different standard called MathML.

In 1997 HTML 4.0 was released as a recommendation of the W3C. HTML 4.0 adopted many specific elements initially developed for a specific web browser, but at the same time began cleaning up the HTML by noting some of them as "deprecated".

HTML 4.0 implements features like XForms 1.0 that do not need to implement navigation engines that were incompatible with some HTML web pages. In 2004 the W3C reopened the discussion of the evolution of HTML, and the foundations for the HTML5 version were unveiled. However, this work was rejected by W3C members and preference would be given to XML development.

Apple, Mozilla and Opera announced their interest in continuing to work on the project under the name of WHATWG, 20 which is based on backward compatibility with previous technologies.

In 2006, W3C became interested in HTML5 development, and in 2007 joined the WHATWG working group to unify the project.

The design in HTML, apart from complying with the specifications of the language, must respect certain web accessibility criteria, following guidelines or the regulations and laws in force in the countries where this concept is regulated. It is available and developed by the W3C through the Web Content Accessibility Guidelines 1.0 WCAG (recently updated with the 2.0 specification), although many countries have their own specifications, as is the case of Spain with the UNE 139803 Standard.
